#  노션 "상태" 프로퍼티의 값을 그룹별로 분류합니다.
#  쉼표(,)로 구분하여 여러 값을 넣을 수 있습니다.
#
#  실행률 계산: (완료 + 지연 완료) / (전체 - 취소 - 참고)

[상태 분류]

# ------------------------------------------------------------
#  1-1. 상태별 의미
#  AI가 각 상태의 맥락을 이해하고 분석에 반영합니다.
# ------------------------------------------------------------

[상태 의미]
완료 = 할 일을 완수함. 기본 완료 상태
Good = 완료했고 스스로 잘했다고 판단한 것
Try = 완료했지만 시도할 점이 있거나 개선이 필요한 것
Problem = 완료했지만 문제가 발생한 것. 원인 분석이 필요
이후 완료 = 당일에 못했지만 나중에라도 완료한 것. 시간 관리 개선 필요
진행 중 = 시작은 했지만 마무리가 안 된 것. 완료로 전환 필요
못함 = 당일 안에 해야 했지만 하지 못한 것. 실패 원인 분석 필요
취소 = 상대방 사정으로 취소된 것 (내 의지 아님). 실행률에서 제외
시작 전 = 아직 시작하지 않은 것. 당일이 지났다면 상태 업데이트가 누락된 것
핀 = 이후에 다시 참고할 메모/레퍼런스. 할 일이 아니므로 실행률에서 제외

#  6. 요일별 점수 기준
#  5가지 항목의 가중치(%)를 설정합니다. 합계는 100이어야 합니다.
#  각 항목은 0~100점으로 측정되고, 가중 평균으로 일일 총점이 나옵니다.
#
#  ① 완료율: 실행 대상 중 완료+지연완료 비율

#  ④ 계획-실행 일치: 계획 시간과 실제 실행 시간의 차이가 적을수록 높은 점수
#  ⑤ 총 실행 시간: 완료한 일의 총 소요 시간이 목표에 가까울수록 높은 점수

[분석 관점]
